# FURNITURE INTAKE OS — PROJECT PROGRESS

**Главная цель:** довести один продаваемый вертикальный продукт для мебельных компаний до работающего пилота.

**Продукт:** сайт + AI-приёмщик + предварительный расчёт + CRM + уведомления + карточка заказа.

**Главный репозиторий продукта:** `Furniture-web-platform-V2`  
**Технологическое ядро:** `furniture-orders-mvp`  
**Первый пилот:** `salamat-mebel.kz`  
**Второй пилот после завершения первого:** `bek-mebel.kz`

### ЭТАП 1 — Доказать полный путь заявки на Salamat Mebel
**Статус:** АКТИВНЫЙ ЭТАП  
**Цель:** один реальный тестовый клиент проходит весь путь без ручного переноса данных между системами.

#### 1.1 Вход клиента
- [ ] На сайте есть один главный CTA: «Рассчитать мебель».
- [ ] CTA открывает форму или AI-диалог.
- [ ] Работает на телефоне.
- [ ] Источник заявки сохраняется.

#### 1.2 Сбор данных

- [ ] Тип мебели.
- [ ] Примерные размеры.

- [ ] Желаемые материалы/цвет.
- [ ] Бюджет.
- [ ] Срок.
- [ ] Адрес или район.
- [ ] Согласие на обработку данных.

#### 1.3 Расчёт
- [ ] Работает хотя бы один калькулятор для одной категории мебели.
- [ ] Формула имеет версию.
- [ ] Цена мебельщика редактируется в настройках.
- [ ] Клиент видит, что расчёт предварительный.
- [ ] Расчёт сохраняется вместе с заявкой.

#### 1.4 CRM
- [ ] Заявка создаётся автоматически.
- [ ] Статус по умолчанию: `Новая`.
- [ ] Видны источник, бюджет, расчёт и заполненность заявки.
- [ ] Менеджер может изменить статус.
- [ ] Менеджер может оставить заметку.

#### 1.5 Уведомления

- [ ] Фиксируется факт ответа менеджера.
- [ ] Логика 10 минут протестирована на тестовых данных.

#### 1.6 Карточка заказа
- [ ] Все собранные данные находятся в одном месте.
- [ ] Приложены фото и файлы.
- [ ] Видна история расчётов.
- [ ] Видна история общения и действий.
- [ ] Можно сформировать краткое ТЗ для замерщика/менеджера.

#### 1.7 Проверка end-to-end
- [ ] Создать тестовую заявку с телефона.
- [ ] Получить расчёт.
- [ ] Убедиться, что заявка появилась в CRM.
- [ ] Получить Telegram-уведомление.
- [ ] Открыть карточку заказа.
- [ ] Изменить статус.
- [ ] Сформировать итоговое краткое ТЗ.
- [ ] Записать обнаруженные ошибки.
- [ ] Исправить критические ошибки.
- [ ] Повторить тест с чистой заявкой.

**Критерий завершения:** две последовательные тестовые заявки проходят весь путь без потери данных и критических ошибок.
